# DeepSeek AI 备份系统管理终端

基于 DeepSeek API 驱动的智能备份系统管理工具，支持文件/数据库备份任务的查询、创建以及自由对话。

---

## 项目结构

```
deepseek-agent/
├── main.go              # 程序入口，菜单驱动交互
├── api.go               # DeepSeek API 客户端封装
├── chat.go              # 自由对话模块
├── file_query.go        # 文件备份任务查询
├── db_query.go          # 数据库备份任务查询
├── file_create.go       # 文件/数据库备份任务创建
├── progress.go          # 进度条模拟（遗留，未使用）
├── go.mod               # Go 模块定义
└── README.md            # 本文件
```

## 各模块详解

### main.go — 程序入口

交互式命令行菜单，使用 `bufio.Scanner` 读取用户输入，根据选项路由到不同的处理函数。

**菜单选项**

| 选项 | 功能 | 说明 |
|------|------|------|
| 1 | 查询文件备份任务 | 向 DeepSeek 请求生成 3 条文件备份任务数据 |
| 2 | 查询数据库备份任务 | 向 DeepSeek 请求生成 3 条数据库备份任务数据 |
| 3 | 创建文件备份任务 | 提交固定参数给 DeepSeek，模拟创建文件备份 |
| 4 | 创建数据库备份任务 | 提交固定参数给 DeepSeek，模拟创建数据库备份 |
| 5 | 自由对话 | 进入多轮对话模式，可问任意问题 |
| 6 | 退出 | 退出程序 |

### api.go — DeepSeek API 客户端

核心 API 封装层，负责与 DeepSeek Chat API 的通信。

**关键配置**
- API Key: 通过环境变量 `DEEPSEEK_API_KEY` 设置，默认使用内置 key
- 模型: 通过环境变量 `DEEPSEEK_MODEL` 设置，默认 `deepseek-v4-pro`
- 超时: 60 秒

**核心函数**

| 函数 | 说明 |
|------|------|
| `chatWithDeepSeek()` | 预设场景使用，system + user 双参数 |
| `chatWithDeepSeekMessages()` | 自由对话使用，传入完整 messages 数组 |
| `callDeepSeek()` | 底层 HTTP 调用，发送请求并解析响应 |
| `printWithReasoning()` | 统一展示：先显示思考过程，再显示最终回答 |

**请求参数**

```json
{
  "model": "deepseek-v4-pro",
  "messages": [...],
  "reasoning_effort": "medium",
  "thinking": {"type": "enabled"},
  "max_tokens": 2048,
  "stream": false
}
```

**响应结构**

API 返回的 `reasoning_content` 字段包含 DeepSeek 的思考过程，是本工具的核心调试功能。

### chat.go — 自由对话

支持多轮对话的交互模块，维护消息历史列表，让 DeepSeek 能够记住上下文。

**特点**
- 输入 `exit` 或 `quit` 返回主菜单
- 保留对话历史，支持连续追问
- 每次回复展示思考过程

### file_query.go — 文件备份任务查询

通过精心设计的 System Prompt 引导 DeepSeek 生成合理的文件备份任务数据。

**System Prompt 要求字段**
- 任务ID、任务名称、服务器名称、服务器IP
- 源路径、目标路径、状态、运行状态
- 调度策略、创建时间、上次执行时间/结果
- 总执行次数、成功/失败次数
- 总数据大小、压缩后大小、文件数量、保留天数

### db_query.go — 数据库备份任务查询

与文件查询类似，引导 DeepSeek 生成数据库备份任务数据。

**System Prompt 要求字段**
- 任务ID、任务名称、数据库类型(MySQL/PostgreSQL/SQL Server)
- 连接地址(host:port)、数据库名称
- 备份类型(全量/增量)、目标路径
- 状态、运行状态、调度策略
- 数据库大小、备份文件大小、保留天数

### file_create.go — 备份任务创建

将用户填写的任务参数拼接成自然语言请求，让 DeepSeek 模拟任务创建确认流程。

**创建文件备份参数**
- 任务名称、服务器名称/IP、源路径、目标路径
- 调度策略、压缩算法、加密算法、保留天数

**创建数据库备份参数**
- 任务名称、数据库类型、连接地址、数据库名
- 备份类型、目标路径、调度策略、压缩/加密、保留天数

### progress.go — 进度条模拟

早期版本的遗留文件，用于在本地假数据模式下模拟备份进度动画。切换到 DeepSeek 驱动模式后不再使用。

## 核心工作流程

```
用户选择菜单操作
       │
       ▼
构造 System Prompt + User Prompt
       │
       ▼
chatWithDeepSeek() → callDeepSeek()
       │
       ▼
POST https://api.deepseek.com/chat/completions
       │
       ▼
解析响应 (content + reasoning_content)
       │
       ▼
printWithReasoning() 输出到终端
   ├── 🤔 思考过程 (reasoning_content)
   └── 最终回答 (content)
```

## 对话演示 (自由对话)

以下为 Terminal 中运行选项 5「自由对话」的完整记录。

### 程序启动

```
╚══════════════════════════════════════════╝
```

### 第一轮对话：问候

**用户**：你好

**DeepSeek 思考过程**：
> 我们被要求用友好的中文回答，帮助用户解决问题。用户说"你好"，我们应该以友好的方式回应，介绍自己是智能助手，询问用户是否需要帮助。

**AI 回答**：
> 你好！很高兴见到你！😊 我是你的智能助手，可以帮你解答问题、找资料、提供建议等等。有什么我可以帮你的吗？

### 第二轮对话：查询黄金价格

**用户**：今天的黄金价格是多少钱每克

**DeepSeek 思考过程**：
> 我们需要理解用户的问题："今天的黄金价格是多少钱每克"。这是一个实时价格查询问题。作为智能助手，我无法直接获取实时数据。...
>
> 由于无法获取实时价格，我需要诚实地告诉用户我无法提供实时数据，但可以给出建议如何查询，或者说明常见价格范围...
>
> 避免给出具体数字。使用"可以搜索"等建议。我会在回复中说明无法提供实时价格，并给出查询途径和注意事项，这样准确有帮助。

**AI 回答**：
> 你好！很抱歉，我目前无法获取实时的黄金价格，因为金价是随时波动的，我没有联网更新即时行情的能力。😣
>
> 不过我可以给你几个查询今日金价的小建议：
> - **直接搜索**：在浏览器搜"今日黄金价格 每克"
> - **常用App/网站**：金投网、同花顺、东方财富等
> - **注意区分价格**：国际金价、国内基础金价、首饰金价各有不同
>
> 如果你告诉我你是想买金饰、投资金条还是了解回收价，我也可以帮你梳理一下参考标准和省钱小技巧～

### 第三轮对话：ZFS 快照原理

**用户**：我问你zfs是如何实现快照的

**DeepSeek 思考过程**（摘要）：
> ZFS快照的核心是写时复制和事务组机制的结合。快照不是立即复制全部数据，而是记录当前元数据指针。创建快照时，ZFS将当前数据集的活动对象集的根指针冻结，并作为快照的对象集。后续写操作通过写时复制分配新块，而快照继续引用旧块。被快照引用的块不会被释放，只有所有快照都删除后空间才回收。

**AI 回答**（概要）：
> ZFS 实现快照的核心是 **写时复制（Copy-On-Write）** 和 **事务组（Transaction Group, TXG）**。回答详细解释了：
>
> 1. **基础结构**：所有数据以"块指针树"形式组织
> 2. **写时复制**：修改时先写新块，再更新指针，不原地修改
> 3. **快照本质**：冻结一份"旧树根"的引用，瞬间完成，几乎不占空间
> 4. **读写过程**：源文件系统通过写时复制生成新树，快照保留旧树
> 5. **事务组保证一致性**：快照基于已提交的事务组，无需 fsck
>
> 最后用"积木搭树"的比喻帮助理解，完整覆盖了原理、机制和优缺点。

**用户**：exit → 返回主菜单 → 选择 6 → 退出程序

```bash

$env:DEEPSEEK_MODEL = "deepseek-v4-pro"

# 启动
go run .
```
